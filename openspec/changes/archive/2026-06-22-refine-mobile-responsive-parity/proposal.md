# Refine Mobile Responsive Parity

## Why

The homepage now closely matches the desktop version of the source site.

However, mobile layouts still differ in spacing, typography, card sizing, content density, and section rhythm.

The goal of this change is to improve the mobile experience while preserving desktop parity and maintaining the existing site architecture.

Unlike the original site, obvious mobile usability issues should be corrected instead of blindly replicated.

## What Changes

- Refine homepage mobile layouts.
- Improve section header responsiveness.
- Improve hero readability on phones.
- Reduce excessive vertical scrolling.
- Improve card density on smaller screens.
- Add an intermediate responsive breakpoint between phone and desktop.
- Improve footer responsiveness.
- Preserve desktop layouts.

## Impact

Affected areas:

- Hero
- About
- Services
- Why Choose Us
- Stats
- News
- Footer

No new routes.

No new dependencies.

No architecture changes.
